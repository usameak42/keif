**Step 6: Create root layout with JS stack and filtration transitions (per D-01, D-02).**

`keif-mobile/app/_layout.tsx`:
Use `@react-navigation/stack` wrapped with `withLayoutContext` from `expo-router`. NOT the native stack (per Pitfall 1 in RESEARCH.md).

The `cardStyleInterpolator` must handle BOTH directions:
- Forward (push): incoming content fades in translating from Y:-40 to Y:0; outgoing fades out translating from Y:0 to Y:+40.
- Back (pop): incoming content fades in from Y:+40 to Y:0; outgoing fades out from Y:0 to Y:-40.

Use `current.progress` and `next.progress` animated nodes. The `next` node is only present when the screen is being covered by a new screen (push), so its presence disambiguates direction.

Duration: 350ms. Easing: ease-out. Card background: `#16100D` (dominant).

Load Inter fonts with `expo-font` and `@expo-google-fonts/inter` (Inter_400Regular, Inter_600SemiBold). Wrap the entire navigator in a font-loading guard that shows nothing until fonts are loaded.

Set `headerShown: false` for all screens.
