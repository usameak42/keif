# Phase 7 — UI Design Contract

> Visual and interaction contract for extended output screens and run history. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Inherits all design tokens from Phase 6 UI-SPEC.

| Property | Value |

## Platform & Navigation

### Extended Navigation Flow

Phase 7 adds screens to the existing three-screen linear flow from Phase 6:

```
[Rotary Selector] --> [Brew Dashboard] --> [Results]
                                              |
                                              +--> [Extended Output] (scroll down or tab)
                                              |
                                              +--> [Run History List]
                                                      |
                                                      +--> [Compare View]
```

- **Extended Output:** Accessed from the Results screen via a "View Details" button below the SCA chart. Pushes a new screen onto the stack with the same filtration-forward transition (D-02).
- **Run History List:** Accessed from the Results screen via a "Save & History" button, or from a persistent "History" touch target on the Results screen header area. Pushes onto the stack.
- **Compare View:** Accessed from Run History List by selecting two runs and tapping "Compare". Pushes onto the stack.
- All new screens use the same filtration transitions (350ms, ease-out, Y translate +/- 40px) defined in Phase 6.
- All new screens include the custom BackButton component from Phase 6.

### New Route Files

```
app/
  extended.tsx      # Extended output detail screen
  history.tsx       # Run history list screen
  compare.tsx       # Side-by-side comparison screen
```

## Screen Contracts

### Screen 4: Extended Output (New)

**Entry:** User taps "View Details" button on Results screen.

**Layout:**
- Scrollable screen, dark background (#16100D).
- Horizontal padding: 32px (xl token).
- Top-left: BackButton ("Back to Results" label, text-secondary, 14px Regular, 44x44px).
- Title: "Detailed Results" (24px SemiBold, text-primary).

**Section A -- Extraction Curve Chart (MOB-09):**
- Card: #251D18 background, border-radius 16px, padding 16px sides / 12px top-bottom.
- Section heading above card: "Extraction Curve" (16px SemiBold, text-primary).
- Victory Native CartesianChart with Line component.
- X-axis: Time (seconds). Y-axis: EY%.
- Line: 2px stroke, #D97A26 (chart-line-primary).
- Axis labels: 14px Regular, #808080.
- Axis ticks: #3B322B.
- Data: `extraction_curve` from SimulationOutput -- array of `{t, ey}` points.
- Chart aspect ratio: 16:9 (wider than SCA chart to show time series).

**Section B -- PSD Curve Chart (MOB-10):**
- Card: same style as extraction curve card.
- Section heading: "Particle Size Distribution" (16px SemiBold, text-primary).
- Victory Native CartesianChart with Line component (area fill optional).
- X-axis: Particle size (um). Y-axis: Volume fraction.
- Line: 2px stroke, #EAE2D7 (chart-psd-line).
- Area fill below line: #EAE2D7 at 10% opacity.
- Data: `psd_curve` from SimulationOutput -- array of `{size_um, fraction}` points.
- Chart aspect ratio: 16:9.

**Section C -- Flavor Axis Chart (MOB-11):**
- Card: same style.
- Section heading: "Flavor Profile" (16px SemiBold, text-primary).
- Three horizontal bars: Sour, Sweet, Bitter.
- Each bar: height 32px, border-radius 8px, fills proportionally (0-1 range mapped to card width minus padding).
- Bar colors: Sour #E8C547, Sweet #6BBF6B, Bitter #C75B39.
- Label left of each bar: "Sour" / "Sweet" / "Bitter" (14px Regular, text-primary).
- Value right of each bar: score as percentage, e.g. "42%" (14px Regular, text-secondary).
- Bars have a track background of #28221F (surface-field).
- Data: `flavor_profile` from SimulationOutput -- `{sour, sweet, bitter}` each 0-1.

**Section D -- Extended Detail Cards (MOB-12):**
- Vertical list of detail cards, each card: #251D18 background, border-radius 12px, padding 16px.
- Gap between cards: 12px.
- Each card has:
  - Field label (14px Regular, #808080) on top line.
  - Field value (16px SemiBold, text-primary) on second line.
  - Optional indicator (color dot or icon) for risk-scored fields.

**Detail card contents (conditional on method and data availability):**

| Field | Label | Value Format | When Shown |
|-------|-------|-------------|------------|
| Channeling Risk | "Channeling Risk" | "{score}/1.0" + colored dot (risk-low/medium/high) | `channeling_risk` is not null (espresso only) |
| CO2 Degassing | "CO2 Degassing Effect" | "{percentage}% suppression" (derived from warnings or bloom data) | Always shown (all methods have bloom window) |
| Water Temp Decay | "Temperature at End" | "{final_temp}C" + mini sparkline or "View curve" link | `temperature_curve` is not null |
| Extraction Uniformity | "Extraction Uniformity" | "{eui}" displayed as fraction + descriptor | `extraction_uniformity_index` is not null |
| Puck Resistance | "Puck Resistance" | "{score}/1.0" | `puck_resistance` is not null (espresso only) |
| Caffeine Estimate | "Caffeine" | "{mg_per_ml} mg/mL" | `caffeine_mg_per_ml` is not null |
| Brew Ratio | "Brew Ratio" | "{ratio}:1" + recommendation text | Always shown |

**EUI descriptor mapping:**
- 0.0-0.3: "Poor" (text-destructive)
- 0.3-0.6: "Moderate" (#E8C547)
- 0.6-0.8: "Good" (text-primary)
- 0.8-1.0: "Excellent" (#6BBF6B)

**Temperature Curve Sub-View:**
- If user taps "View curve" on the temperature card, expand inline to show a Victory Native line chart.
- Chart: X-axis Time (s), Y-axis Temperature (C).
- Line: 2px stroke, #EB9E47 (chart-temp-line).
- Chart fits inside the same card (card expands to accommodate), aspect ratio 3:1 (compact).

### Screen 5: Run History List (New)

**Entry:** User taps "Save & History" on Results screen (after a simulation is complete).

**Save Flow (MOB-13):**
- When entering history screen for the first time after a simulation, a save prompt appears at the top:
  - Card: #251D18 background, border-radius 12px, padding 16px, border-left 4px solid #D97A26.
  - Heading: "Save This Run?" (16px SemiBold, text-primary).
  - Text input for run name: surface-field background (#28221F), border 1px #3B322B, 48px height, placeholder "e.g. Morning V60" (#808080).
  - "Save" button: accent background (#D97A26), 44px height, full-width, text "#16100D" 16px SemiBold.
  - "Skip" text below button: 14px Regular, #808080, 44x44px touch target.
- If the current simulation result is already saved, the save prompt does not appear.

**Layout (MOB-14):**
- Scrollable screen, dark background.
- Horizontal padding: 32px (xl).
- Top-left: BackButton ("Back" label).
- Title: "Run History" (24px SemiBold, text-primary).
- Below title: Run count badge, e.g. "23 runs" (14px Regular, #808080).

**Run List:**
- FlatList of saved runs, sorted by date (newest first).
- Each list item: #251D18 background, border-radius 12px, padding 16px, height 72px.
- Gap between items: 8px (sm).
- Left section: Run name (16px Regular, text-primary) on first line. Method + date below (14px Regular, #808080), e.g. "V60 -- Mar 28, 2026".
- Right section: TDS% and EY% summary (14px SemiBold, text-primary), stacked vertically: "TDS 1.35%" / "EY 20.1%".
- Selected state (for comparison): accent border (2px solid #D97A26), background shifts to #28221F.

**Selection for Comparison (MOB-15):**
- Long-press on a list item enters selection mode.
- In selection mode: each item shows a circular checkbox on the left (24px diameter, #3B322B border, #D97A26 fill when selected).
- User can select exactly 2 runs.
- When 2 are selected, a "Compare" button appears at the bottom: full-width, accent background (#D97A26), 52px height, text "Compare" (#16100D, 16px SemiBold).
- When fewer or more than 2 are selected, button shows "Select 2 runs to compare" (disabled state: #3B322B background, #808080 text).
- Tap on a non-selected item in selection mode toggles its checkbox.
- Tap "Cancel" (14px Regular, #808080, top-right) to exit selection mode.

**Archive Prompt (MOB-16):**
- When saved runs exceed 100, a banner appears at the top of the list (below title, above save prompt):
  - Background: #E8C547 at 15% opacity on #251D18 base.
  - Border-left: 4px solid #E8C547.
  - Icon: Ionicons "alert-circle" in #E8C547, 20px.
  - Text: "You have {count} saved runs. Archive older runs to keep the app fast." (14px Regular, text-primary).
  - Two actions:
    - "Archive Runs Older Than 30 Days" -- outlined button (1px border #E8C547, transparent bg, #E8C547 text, 14px SemiBold, 40px height).
    - "Dismiss" -- text button (14px Regular, #808080).
- Archive action: soft-delete (mark as archived in SQLite, remove from main list, keep data).

**Swipe to Delete (single run):**
- Swipe left on a list item reveals a delete action area: #D94F4F background, "Delete" text (#EAE2D7, 14px SemiBold), Ionicons "trash-outline" icon.
- Confirmation: bottom modal -- "Delete this run? This cannot be undone." (16px Regular, text-primary).
  - "Delete" button: #D94F4F background, #EAE2D7 text, 16px SemiBold, 48px height.
  - "Cancel" button: outlined, 1px border #3B322B, text-secondary, 16px Regular, 48px height.

**Empty State:**
- When no saved runs exist:
  - Centered vertically in available space.
  - Icon: Ionicons "flask-outline" in #808080, 48px.
  - Heading: "No Saved Runs" (16px SemiBold, text-primary).
  - Body: "Run a simulation and save it here to track your brews over time." (14px Regular, #808080, text-align center, max-width 280px).

### Screen 6: Compare View (New)

**Entry:** User taps "Compare" on Run History after selecting 2 runs.

**Layout:**
- Scrollable screen, dark background.
- Horizontal padding: 32px (xl) reduced to 16px (md) for comparison columns to maximize width.
- Top-left: BackButton ("Back" label).
- Title: "Compare Runs" (24px SemiBold, text-primary).
- Below title: Run A name vs Run B name (14px Regular, #808080), separated by "vs".

**Section A -- Key Metrics Side-by-Side (MOB-15):**
- Two columns, 8px (sm) gap between them.
- Each column is a card: #251D18 background, border-radius 12px, padding 12px.
- Column header: Run name (14px SemiBold, text-primary, truncated with ellipsis if > 12 chars).
- Metrics in each column:
  - TDS%: 24px SemiBold, text-primary. Label "TDS" 14px Regular, #808080.
  - EY%: 24px SemiBold, text-primary. Label "EY" 14px Regular, #808080.
  - Zone verdict: 14px SemiBold, color mapped to zone (same as Phase 6 ZoneVerdict).
- Difference indicator between columns for TDS and EY: delta shown in accent color if positive, text-secondary if negative, e.g. "+0.12%" or "-1.3%".

**Section B -- Overlaid Extraction Curves (MOB-15):**
- Single chart card: #251D18 background, border-radius 16px, padding 16px.
- Section heading: "Extraction Curves" (16px SemiBold, text-primary).
- Victory Native CartesianChart with two Line components overlaid.
- Run A line: 2px stroke, #D97A26 (chart-line-primary).
- Run B line: 2px stroke, #5B9BD5 (chart-line-secondary), dashed (dashArray [6, 4]).
- Legend below chart: two items, each with colored line sample (16px wide, 2px height) + run name (14px Regular, text-primary). Gap between legend items: 16px (md).
- X-axis: Time (seconds). Y-axis: EY%.
- Chart aspect ratio: 16:9.

**Section C -- SCA Chart with Both Points (MOB-15):**
- Same SCA chart style from Phase 6 (ideal zone rectangle + scatter points).
- Run A point: filled circle, 10px, #D97A26.
- Run B point: filled circle, 10px, #5B9BD5, with 1px #EAE2D7 border for distinction.
- Legend below chart: same pattern as extraction curves legend.

**Section D -- Flavor Comparison (MOB-15):**
- Section heading: "Flavor Profile" (16px SemiBold, text-primary).
- Three grouped horizontal bar pairs (Sour, Sweet, Bitter):
  - Each group has two bars stacked vertically (Run A on top, Run B below).
  - Run A bar uses flavor fill colors from Phase 7 color table (Sour #E8C547, Sweet #6BBF6B, Bitter #C75B39).
  - Run B bar uses the same color at 50% opacity.
  - Label left: flavor name (14px Regular, text-primary).
  - Values right of each bar: Run A value / Run B value (14px Regular, text-secondary).
  - Bar height: 24px each, gap between A and B: 4px (xs), gap between flavor groups: 16px (md).

| Hook | Purpose |

- Archive prompt triggers at 100 active (non-archived) runs.

## API Integration Contract (Phase 7 additions)

### SimulationOutput Type Extension

The TypeScript `SimulationOutput` interface in `types/simulation.ts` must be extended to include all fields the API already returns:

```typescript
export interface ExtractionPoint {
  t: number;       // seconds
  ey: number;      // extraction yield %
}

export interface PSDPoint {
  size_um: number;  // particle size um
  fraction: number; // volume fraction 0-1
}

export interface FlavorProfile {
  sour: number;     // 0-1
  sweet: number;    // 0-1
  bitter: number;   // 0-1
}

export interface TempPoint {
  t: number;        // seconds
  temp_c: number;   // degrees C
}

export interface SimulationOutput {
  tds_percent: number;
  extraction_yield: number;
  extraction_curve: ExtractionPoint[];
  psd_curve: PSDPoint[];
  flavor_profile: FlavorProfile;
  brew_ratio: number;
  brew_ratio_recommendation: string;
  warnings: string[];
  mode_used: string;
  sca_position: SCAPosition | null;
  channeling_risk: number | null;
  extraction_uniformity_index: number | null;
  temperature_curve: TempPoint[] | null;
  puck_resistance: number | null;
  caffeine_mg_per_ml: number | null;
}
```

No new API endpoints needed. Phase 7 uses the same POST /simulate response; it simply renders more fields from the existing SimulationOutput.

- Screen width assumption: 360-428px.

Phase 7 specifics:

## Accessibility

Inherited from Phase 6 (minimum 44x44px touch targets, accessible labels, color never sole indicator).

Phase 7 additions:
- Charts: each chart card has `accessibilityLabel` summarizing the data (e.g., "Extraction curve: EY rises from 0% to 20.1% over 240 seconds").
- Flavor bars: `accessibilityLabel` per bar (e.g., "Sour: 42 percent").
- Run history items: `accessibilityLabel` includes run name, method, date, TDS, and EY.
- Selection mode: `accessibilityRole="checkbox"` on selection checkboxes with `accessibilityState={{ checked }}`.
- Delete confirmation modal: `accessibilityViewIsModal={true}` to trap focus.
- Comparison delta values: include direction in label (e.g., "TDS difference: plus 0.12 percent").
- Archive banner: `accessibilityRole="alert"`.
