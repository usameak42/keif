## Spacing Scale

Inherited from Phase 6 without modification. All tokens from `constants/spacing.ts` apply:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between pills in scroll row |
| sm | 8px | Horizontal padding inside each pill |
| md | 16px | Gap between roast selector and adjacent form rows; label margin-bottom |
| lg | 24px | — |
| xl | 32px | Screen horizontal padding (unchanged from dashboard) |
| xxl | 48px | Space above Run Simulation button |
| xxxl | 64px | — |

Phase 8 exceptions:
- Pill internal horizontal padding: 12px left + 12px right (between xs and md — not a new token, use literal 12). This gives "Medium Light" and "Medium Dark" enough room at 14px font without truncation on 360px screens.
- Pill height: 36px (one step below the existing 40px SegmentedControl). Rationale: 5 pills are scroll-accessed, not tap-and-compare; the reduced height keeps the row compact without affecting the 44px scroll touch area.
- Pill scroll row minimum touch height: 44px (containing ScrollView, not the pill itself). Use `style={{ paddingVertical: 4 }}` on the ScrollView to reach 44px total interactive height.

## Component Contract: RoastPillSelector

### Component Name

`RoastPillSelector` — file: `components/RoastPillSelector.tsx`

### Interface

```typescript
interface RoastPillSelectorProps {
  selectedLevel: RoastLevel;
  onSelect: (level: RoastLevel) => void;
}
```

Where `RoastLevel = "light" | "medium_light" | "medium" | "medium_dark" | "dark"` (updated type from D-11).

### Pill Configuration

| Index | API Value | Display Label |
|-------|-----------|---------------|
| 0 | `"light"` | `"Light"` |
| 1 | `"medium_light"` | `"Medium Light"` |
| 2 | `"medium"` | `"Medium"` |
| 3 | `"medium_dark"` | `"Medium Dark"` |
| 4 | `"dark"` | `"Dark"` |

Labels use SCA standard terminology (D-09). Labels are title-case. No abbreviations. Do NOT use "Med-Light" or "Med-Dark".

### Layout

- Container: `ScrollView` with `horizontal={true}`, `showsHorizontalScrollIndicator={false}`, `contentContainerStyle={{ gap: 4 }}` (xs token).
- Pills are laid out in a single horizontal row. They do NOT use `flex: 1` (no flex-grow). Each pill sizes to its content width.
- The row is left-aligned. Scroll is triggered naturally when content overflows.
- On a 360px screen (minimum target): all 5 pills overflow the available width and scroll is required. On wider screens (428px), overflow still occurs. Horizontal scroll is always available and expected.
- No snap behavior. Free scroll. On mount, after AsyncStorage restores the selected level, scroll the selected pill into view using a `ScrollView` ref + `scrollTo`. The pill must be visible (not necessarily centered) — use the minimum scroll distance to make it visible. If the selected level is "light" or "medium", no scroll is needed (already in view). If "medium_dark" or "dark", scroll right until the pill is fully visible. Implement via `onLayout` on the pill container plus a `ref.current.scrollTo({ x: offsetX, animated: false })` call after the async read completes. No animated scroll — instant snap to position.

### Pill Dimensions

- Height: 36px.
- Horizontal padding: 12px left, 12px right.
- Width: content-sized (no fixed width). "Medium Light" at 14px ≈ 80px glyph width + 24px padding = ~104px per pill. "Light" and "Dark" ≈ 60px.
- Border radius: 20px.
- No minimum width constraint.

### Pill Transitions

- Background color change on select: no animation. Instant color swap. Rationale: matches the existing `SegmentedControl` behavior (no animation in Phase 6 spec; 200ms ease-out was for the entire segment row, not individual pills). Keeps implementation simple.
- Pressed state: via `Pressable` with `android_ripple` disabled. Use `style={({ pressed }) => [...]}` pattern (same as existing components in the codebase).

### Accessibility

- `ScrollView` wrapper: `accessibilityRole` not set (it is a scrollable container, not a widget).
- Each `Pressable` pill: `accessibilityRole="radio"`, `accessibilityState={{ selected: isSelected }}`, `accessibilityLabel="{label}"` (e.g., `"Light"`, `"Medium Light"`).
- The group as a whole should be preceded by the "Roast Level" section label, which provides context.
- Touch target: the `Pressable` itself is 36px tall. The containing `ScrollView` adds 4px padding vertically (2px top, 2px bottom) reaching 40px total. This is below the 44px minimum guideline. Mitigate by setting `hitSlop={{ top: 4, bottom: 4 }}` on each `Pressable` to pad to 44px effective touch target without changing visual size.

## Dashboard Integration

### Section Order (unchanged from Phase 6 D-07)

The `RoastPillSelector` replaces the `SegmentedControl` for roast level in `dashboard.tsx`. All other parameter fields remain in the same order:

| # | Field | Component |
|---|-------|-----------|
| 1 | Grinder Model | GrinderDropdown (unchanged) |
| 2 | Grinder Setting / Grind Size | ClickSpinner or FormField (unchanged) |
| 3 | Dose | FormField (unchanged) |
| 4 | Water | FormField (unchanged) |
| 5 | Temperature | FormField (unchanged) |
| 6 | Time | FormField (unchanged) |
| **7** | **Roast Level** | **RoastPillSelector (replaces SegmentedControl)** |
| 8 | Mode | SegmentedControl (unchanged — Fast / Accurate stays as segmented control) |
| 9 | Run Simulation | SimulateButton (unchanged) |

Section label "Roast Level" above the selector: `Typography.label` style, `Colors.textSecondary`, `marginBottom: Spacing.sm` (8px). This is identical to all other section labels in the dashboard.

Field gap wrapper: `marginBottom: Spacing.md` (16px) — same as all other form rows.

### Default Selection on Mount

1. On mount, read `AsyncStorage.getItem("lastRoastLevel")`.
2. If value is a valid `RoastLevel` string: set that as the initial selection.
3. If null or invalid: default to `"medium"` (index 2).
4. On each `onSelect` call: write `AsyncStorage.setItem("lastRoastLevel", level)`.
5. The AsyncStorage read is async. During the read, pre-select `"medium"` as interim state. Replace with stored value when read completes (no visible flash expected; read is sub-10ms on device).

AsyncStorage pattern: use a `useEffect` on mount inside `dashboard.tsx`, not inside `RoastPillSelector` itself. The component receives `selectedLevel` as a prop (controlled). State and persistence live in the screen. This follows the pattern already used in the dashboard for other controlled form fields.

## Responsive Behavior

Inherited from Phase 6:
- Target: mobile phones only (iOS + Android).
- Screen width assumption: 360–428px.

Phase 8 specifics:
- At 360px: available scroll content width for pills ≈ 360 − 64 (horizontal padding) = 296px. Total pill row content width ≈ 5 pills × ~80px average + 4 gaps × 4px = ~416px. Overflow: ~120px. Scroll required and expected.
- At 428px: available = 364px. Overflow: ~52px. Scroll still required and expected. Do NOT attempt to fit pills without scroll on any supported screen width.
- Pills do not wrap to a second line. Single horizontal row only.
- `ScrollView` must have `horizontal={true}` and `nestedScrollEnabled={true}` to avoid scroll conflict with the parent vertical `ScrollView` on the dashboard.
